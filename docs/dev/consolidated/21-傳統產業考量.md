# 傳統產業考量

## 傳統產業的特性

### 與軟體產業的差異

**軟體產業:**
```
特性:
- 純數位化
- 快速迭代
- 遠端協作
- 即時修改
- 低邊際成本

工作方式:
- 敏捷開發
- 持續交付
- 自動化測試
- 雲端部署
```

**傳統產業 (工程、製造):**
```
特性:
- 實體作業為主
- 長週期專案
- 現場執行
- 修改成本高
- 高邊際成本

工作方式:
- 詳細規劃先行
- 階段性交付
- 人工檢驗為主
- 現場安裝
```

### 現場優先的設計理念

**核心原則: 系統服務現場,而非控制現場**

**情境 A: 計畫與現實的落差**
```
早上規劃:
  Task-001: SS-A01 配線 [張三組 3人]
  預估: 8 小時完成

下午現場:
  實際狀況:
    - 王五請假,只有 2 人
    - 發現規格不符,需要確認
    - 下雨影響作業
    
系統應該:
  ✓ 允許調整人數
  ✓ 允許新增緊急 Task
  ✓ 允許延後完成時間
  
系統不應該:
  ✗ 強制照原計畫執行
  ✗ 拒絕變更
  ✗ 過度複雜的流程
```

**情境 B: 主管的現場判斷**
```
Process 建議: "接下來應該配線"
主管判斷: "材料還沒到,先做別的"

系統應該:
  ✓ 記錄偏離原因
  ✓ 尊重主管判斷
  ✓ 提供替代建議
  
系統不應該:
  ✗ 強制照 Process 走
  ✗ 阻擋主管決策
  ✗ 產生錯誤訊息
```

## 人力資源管理

### 人員流動性高

**挑戰:**
```
傳統產業常見:
  - 臨時工、約聘工
  - 師傅帶徒弟制
  - 人員流動率高
  - 跨公司協作
  
範例:
  張三組: 3 人 (今天)
  張三組: 2 人 (明天,王五請假)
  張三組: 4 人 (後天,加入臨時工)
```

**系統設計:**
```typescript
// 靈活的人員記錄
interface TaskExecution {
  taskId: string;
  
  // 不強制固定人員名單
  assignedTeam: string;  // "張三組" (團隊名)
  
  // 記錄實際出工
  actualParticipants: Participant[];
}

interface Participant {
  name: string;
  role: string;
  workHours: number;
  date: Date;
  
  // 允許臨時人員
  isTemporary?: boolean;
  employmentType?: 'Permanent' | 'Contract' | 'Temporary';
}
```

### 技能與專業分工

**多專業協作:**
```
一個機櫃安裝需要:
  - 吊裝專業: 搬運定位
  - 機電專業: 內部組裝
  - 配線專業: 配線施工
  - 儀控專業: 測試調試
  
特性:
  - 不同公司
  - 不同專長
  - 順序依賴
  - 交接點多
```

**系統設計:**
```typescript
// 記錄專業需求
interface TaskTemplate {
  title: string;
  
  // 需要的專業
  requiredSkills: string[];
  
  // 可執行的團隊
  eligibleTeams: string[];
  
  // 前置技能
  prerequisiteSkills?: string[];
}
```

## 實體作業考量

### 物料與設備

**物料管理:**
```
挑戰:
  - 材料進場時間不確定
  - 規格可能不符
  - 數量可能短缺
  - 存放空間有限
  
影響:
  Task 可能因為材料未到而阻擋
```

**設備調度:**
```
挑戰:
  - 吊車、工具車共用
  - 需要提前預約
  - 可能臨時故障
  - 天氣影響使用
  
影響:
  Task 排程需要考慮設備可用性
```

**系統設計:**
```typescript
// 資源依賴
interface TaskDependency {
  taskId: string;
  
  // 物料依賴
  requiredMaterials: Material[];
  
  // 設備依賴
  requiredEquipment: Equipment[];
  
  // 檢查可用性
  checkAvailability(): Promise<AvailabilityStatus>;
}

interface Material {
  materialCode: string;
  quantity: number;
  
  // 到貨狀態
  deliveryStatus: 'Ordered' | 'InTransit' | 'Delivered';
  expectedDeliveryDate?: Date;
}
```

### 天氣與環境因素

**天氣影響:**
```
下雨天:
  - 不能室外配線
  - 吊車無法作業
  - 某些材料不能淋雨
  
高溫天:
  - 戶外作業時間縮短
  - 人員需要更多休息
  - 某些材料施工品質受影響
```

**系統設計:**
```typescript
// 環境條件
interface EnvironmentalCondition {
  date: Date;
  location: string;
  
  weather: 'Sunny' | 'Rainy' | 'Windy' | 'Hot' | 'Cold';
  temperature: number;
  
  // 影響的 Tasks
  affectedTasks: string[];
  
  // 建議
  recommendations: string[];
}

// Event 記錄
Event: TaskPostponedDueToWeather
  taskId: "task-001"
  originalDate: "2024-03-15"
  newDate: "2024-03-16"
  reason: "下雨,無法室外配線"
  weatherCondition: { weather: "Rainy", ... }
```

## 安全與合規

### 工安要求

**強制性規定:**
```
傳統產業必須遵守:
  - 職業安全衛生法
  - 高風險作業許可
  - 個人防護裝備
  - 安全教育訓練
  
範例:
  高空作業:
    - 需要工作許可
    - 雙人作業制
    - 安全帶、安全網
    - 現場監工
```

**系統設計:**
```typescript
// 安全檢查
interface SafetyCheck {
  taskId: string;
  
  // 風險等級
  riskLevel: 'Low' | 'Medium' | 'High' | 'Critical';
  
  // 需要的許可
  requiredPermits: Permit[];
  
  // 防護裝備
  requiredPPE: string[];
  
  // 安全教育
  requiredTraining: string[];
}

// Event 記錄
Event: SafetyCheckPerformed
  taskId: "task-001"
  checkDate: "2024-03-15 07:30:00"
  checkedBy: "工安人員"
  checkResult: "Passed"
  checklist: [
    { item: "工作許可", status: "OK" },
    { item: "安全帶", status: "OK" },
    { item: "安全網", status: "OK" }
  ]
```

### 品質要求

**驗收標準:**
```
傳統產業驗收:
  - 實體檢查 (目視、測量)
  - 功能測試
  - 文件檢查
  - 客戶簽核
  
不像軟體:
  - 自動化測試
  - 即時驗證
  - 容易回滾
```

**系統設計:**
```typescript
// 支援人工驗收
interface ManualInspection {
  taskId: string;
  
  // 檢查項目
  checklistItems: ChecklistItem[];
  
  // 測量記錄
  measurements: Measurement[];
  
  // 照片證據
  photos: Photo[];
  
  // 簽核
  inspectorSignature: string;
  clientSignature?: string;
}
```

## 文化與習慣適應

### 數位化程度

**現實狀況:**
```
現場人員:
  - 年齡層廣 (20-60歲)
  - 數位素養差異大
  - 習慣紙本記錄
  - 可能沒有智慧手機
  
挑戰:
  - 不能要求全數位化
  - 需要漸進式轉型
  - 要有紙本備援
```

**系統設計:**
```
混合模式:
  - 提供 App,但不強制
  - 支援紙本轉數位
  - 簡化 UI,大按鈕
  - 語音輸入支援
  - 離線模式
  
範例:
  現場主管:
    - 早上用紙本分配工作
    - 晚上回辦公室輸入系統
    - 系統自動產生 Events
```

### 溝通方式

**傳統溝通:**
```
現場習慣:
  - 面對面口頭溝通
  - 電話聯絡
  - 即時決策
  - 簡訊/LINE 群組
  
不習慣:
  - Email 正式流程
  - 系統工單
  - 等待審核
```

**系統設計:**
```
靈活通知:
  - 多種通知方式
    - App 推播
    - 簡訊
    - Email
    - LINE/WeChat
  
  - 即時與非即時混合
    - 緊急: 電話 + 簡訊
    - 一般: App 推播
    - 記錄: Email
```

## 總結

傳統產業系統設計的核心:
1. **現場優先** - 尊重現場判斷
2. **彈性調整** - 允許計畫變更
3. **實體考量** - 物料、設備、天氣
4. **安全合規** - 工安、品質要求
5. **漸進數位** - 不強制全數位化
6. **混合溝通** - 多種通知方式
